# Исправление обрезания выпадающих окон фильтров

## Проблема
Выпадающие окна Excel-фильтров обрезались контейнером таблицы из-за `overflow: hidden` или `overflow: auto` на родительских элементах.

## Решение
Использован **React Portal** для рендера выпадающих окон напрямую в `document.body`, что позволяет им отображаться поверх всех элементов страницы.

## Что изменилось

### ExcelFilter.js

**Добавлено:**
1. Импорт `ReactDOM` для создания Portal
2. Новый ref `dropdownRef` для выпадающего окна
3. Состояние `position` для хранения координат окна
4. Обработчики скролла и ресайза для обновления позиции

**Изменено:**
- Позиционирование: с `position: absolute` внутри родителя на `position: absolute` в `document.body`
- z-index: увеличен до 10000 для отображения поверх всех элементов
- Клик вне области: обновлен для работы с Portal

## Технические детали

### React Portal
```javascript
{isOpen && ReactDOM.createPortal(
  <div ref={dropdownRef} style={{ ... }}>
    {/* Содержимое фильтра */}
  </div>,
  document.body  // Рендерим в body
)}
```

### Вычисление позиции
```javascript
const rect = buttonRef.current.getBoundingClientRect();
setPosition({
  top: rect.bottom + window.scrollY + 4,
  left: rect.left + window.scrollX
});
```

### Обработка событий
- **Скролл**: обновляет позицию окна при прокрутке страницы
- **Resize**: обновляет позицию при изменении размера окна
- **Click outside**: закрывает окно при клике вне кнопки и окна

## Преимущества решения

✅ **Выпадающее окно не обрезается** - рендерится в body
✅ **Следует за кнопкой** - позиция обновляется при скролле
✅ **Высокий z-index** - отображается поверх всех элементов
✅ **Корректное закрытие** - работает клик вне области
✅ **Адаптивность** - реагирует на resize окна
